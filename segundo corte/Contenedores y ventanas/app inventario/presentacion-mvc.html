¿Qué es MVC?

Es un patrón que divide la aplicación en tres capas con responsabilidades distintas. Cada capa hace una sola cosa y no se mete en el trabajo de las otras.

🗄️

Model

Representa los datos y sabe cómo guardarlos, leerlos y modificarlos.

⚙️

Controller

Contiene la lógica del negocio. Valida y decide qué hacer con los datos.

🖥️

View

Es lo que ve el usuario. Muestra datos y captura acciones (clics, texto).

Estructura del proyecto

app inventario/ ├── views/ │ └── Productos.cs ← VIEW │ ├── controllers/ │ └── Productos.cs ← CONTROLLER │ ├── models/ │ └── Producto.cs ← MODEL │ └── bd/ └── productos.csv ← Datos persistidos

Cada carpeta = una capa. El nombre del archivo es el mismo en las tres capas a propósito.

MODEL

models/Producto.cs

Representa un producto y sabe cómo persistir los datos en el archivo CSV.

  • Define las propiedades del objeto
  • Crea una línea en el CSV
  • Lee todas las líneas del CSV
  • Busca por ID
  • Actualiza y elimina líneas

El Model no valida, no muestra mensajes.
Solo trabaja con los datos.

models/Producto.cs
public class Producto { private const string RUTA = "../db/productos.csv"; public int Id { get; set; } public string Descripcion { get; set; } public int Precio { get; set; } public void Crear(int id, string desc, int precio) { File.AppendAllText(RUTA, $"{id},{desc},{precio}\n"); } public List<Producto> Listar() { // Lee el CSV y retorna la lista } public Producto Buscar(int id) { ... } public void Actualizar(...) { ... } public void Eliminar(int id) { ... } }
CONTROLLER

controllers/Productos.cs

Contiene la lógica del negocio. Antes de tocar el Model, valida que todo esté bien.

  • ¿Ya existe ese ID?
  • ¿El producto existe antes de eliminarlo?
  • Retorna "ok" o un mensaje de error
  • La View solo decide qué hacer con ese resultado

El Controller no sabe nada de botones ni grids.
Solo recibe datos, valida y responde.

controllers/Productos.cs
public class Productos { public string Crear(int id, string desc, int precio) { Producto p = new Producto(); if (p.Buscar(id) != null) return "El producto con id " + id + " ya existe."; p.Crear(id, desc, precio); return "ok"; } public List<Producto> Listar() { return new Producto().Listar(); } public string Eliminar(int id) { ... } public string Actualizar(...) { ... } }
VIEW

views/Productos.cs

Es el Windows Form. Captura lo que el usuario escribe, llama al Controller y muestra el resultado.

  • Instancia el Controller
  • Al abrir el form, carga la lista
  • Al hacer clic en Guardar, llama a Crear
  • Si el resultado es "ok", recarga el grid
  • Si hay error, muestra un MessageBox

La View no valida ni toca el CSV.
Solo se comunica con el Controller.

views/Productos.cs
public partial class Productos : Form { private controllers.Productos _controller = new controllers.Productos(); private void Productos_Load(...) { CargarProductos(); } private void CargarProductos() { dataGridView1.DataSource = _controller.Listar(); } private void button1_Click(...) // Guardar { string resultado = _controller.Crear(id, desc, precio); if (resultado == "ok") { CargarProductos(); LimpiarCampos(); } else { MessageBox.Show(resultado); } } }

Resumen

🗄️

Model

models/Producto.cs

Propiedades + CRUD sobre el CSV. No valida, solo persiste datos.

⚙️

Controller

controllers/Productos.cs

Lógica y validaciones. Retorna "ok" o un error. No sabe de la UI.

🖥️

View

views/Productos.cs

Windows Form. Captura eventos, llama al Controller y actualiza la UI.

Regla de oro: cada capa habla solo con la capa de al lado. La View no toca el CSV. El Model no muestra MessageBox.
